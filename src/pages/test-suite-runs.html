Test Suite Runs

What is a test suite run?

A test suite run executes your RSpec test suite against a specific commit. SaturnCI automatically triggers test suite runs when you push to GitHub, but you can also trigger them programmatically using the SDK.

Automatic test suite runs

Once your repository is set up with SaturnCI, test suite runs are triggered automatically on every push to GitHub. No additional configuration is needed.

Triggering a test suite run via the SDK

You can also trigger test suite runs programmatically using the SaturnCI SDK: 

require 'saturnci-sdk'

client = SaturnCI::Client.new

test_suite_run = SaturnCI::TestSuiteRun.create(
  client: client,
  repository: 'your-org/your-repo',
  branch_name: 'main',
  commit_hash: `git rev-parse HEAD`.strip,
  commit_message: `git log -1 --format=%s`.strip,
  author_name: `git log -1 --format=%an`.strip
)

puts "Testing: #{test_suite_run.url}"
test_suite_run.wait_for_completion
puts "Status: #{test_suite_run.status}"

Test, build, and deploy

Test suite runs can be combined with builds and jobs to create a full test-build-deploy pipeline: 

require 'saturnci-sdk'

client = SaturnCI::Client.new

# Test
test_suite_run = SaturnCI::TestSuiteRun.create(
  client: client,
  repository: 'your-org/your-repo',
  branch_name: 'main',
  commit_hash: `git rev-parse HEAD`.strip,
  commit_message: `git log -1 --format=%s`.strip,
  author_name: `git log -1 --format=%an`.strip
)
puts "Testing: #{test_suite_run.url}"
test_suite_run.wait_for_completion
abort "Tests failed!" unless test_suite_run.status == "Passed"

# Build
build = SaturnCI::Build.create(
  client: client,
  repository: 'your-org/your-repo',
  name: 'production'
)
puts "Building: #{build.url}"
build.wait_for_completion
puts "Image: #{build.container_image_url}"

# Deploy
job = SaturnCI::Job.create(
  client: client,
  repository: 'your-org/your-repo',
  name: 'deploy',
  container_image_url: build.container_image_url
)
puts "Deploying: #{job.url}"
job.wait_for_completion
puts "Deploy complete!"
