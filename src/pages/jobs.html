Jobs

What is a job?

A job is an arbitrary task that SaturnCI runs in a Docker container. Jobs can be used for deployments, database migrations, or any other scripted task you want to run in a reproducible environment.

Setting up a job

To set up a job, create a directory at .saturnci/jobs/<job-name>/ in your repository with three files:

  • Dockerfile — the Docker image to run the job in
  • docker-compose.yml — the Docker Compose configuration
  • run — the script that gets executed

Example: a "hello world" job

.saturnci/jobs/hello_world/Dockerfile:

FROM ruby:3.3.0-slim

.saturnci/jobs/hello_world/docker-compose.yml:

services:
  job:
    build:
      context: ../../..
      dockerfile: .saturnci/jobs/hello_world/Dockerfile
    volumes:
      - ../../../:/app
    working_dir: /app

.saturnci/jobs/hello_world/run:

#!/bin/sh
echo "hello world"

Make sure the run script is executable: 

chmod +x .saturnci/jobs/hello_world/run

Commit these files and push to your repository. You can then trigger the job using the SaturnCI SDK: 

require 'saturnci-sdk'

client = SaturnCI::Client.new

job = SaturnCI::Job.create(
  client: client,
  repository: 'your-org/your-repo',
  name: 'hello_world'
)

puts "Running: #{job.url}"
job.wait_for_completion
puts "Done! Status: #{job.status}"
